## 1. 服务器部署

### 1.1 部署架构

```
┌─────────────────────────────────────────────────────────┐
│                     Linux 服务器                         │
│                                                         │
│  ┌──────────────┐    ┌──────────────┐                  │
│  │   Nginx      │    │    PM2       │                  │
│  │  (前端静态)   │    │ (后台服务)   │                  │
│  │  80/443端口   │    │  9998端口    │                  │
│  └──────┬───────┘    └──────┬───────┘                  │
│         │                   │                           │
│         ▼                   ▼                           │
│  ┌──────────────┐    ┌──────────────┐                  │
│  │  /www/wwwroot│    │ /www/firefly │                  │
│  │   /firefly   │    │   /admin     │                  │
│  │  (构建产物)   │    │  (源码+构建)  │                  │
│  └──────────────┘    └──────────────┘                  │
│                                                         │
│  ┌──────────────────────────────────────────────────┐   │
│  │  /www/firefly/ (源码目录)                         │   │
│  │    ├── src/        (前端源码)                     │   │
│  │    ├── admin/      (后台源码)                     │   │
│  │    ├── dist/       (前端构建产物)                  │   │
│  │    ├── deploy.sh   (一键部署脚本)                   │   │
│  │    └── package.json                               │   │
│  └──────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
```

**关键决策：部署源码而非仅构建产物**

- ✅ **优点**：支持 `git pull` 更新、无需本地构建、后台一键更新
- ✅ **优点**：内容变更（文章、配置）可直接通过 git 同步
- ✅ **优点**：服务器端构建，保证环境一致性
- ⚠️ **注意**：需要在服务器安装 Node.js 和 pnpm

#### 步骤 1：安装宝塔面板

### 1.4 配置 Nginx

在宝塔面板中添加网站：

1. **域名设置**：填写你的域名（如 `blog.example.com`）
2. **根目录**：设置为 `/www/wwwroot/firefly`
3. **SSL 证书**：申请并配置 SSL（推荐 Let's Encrypt）

#### Nginx 配置示例

```nginx
server {
    listen 80;
    listen 443 ssl;
    server_name blog.example.com;
    
    # SSL 配置
    ssl_certificate /www/server/panel/vhost/cert/blog.example.com/fullchain.pem;
    ssl_certificate_key /www/server/panel/vhost/cert/blog.example.com/privkey.pem;
    
    # 根目录
    root /www/wwwroot/firefly;
    index index.html;
    
    # 前端路由重写（SPA 需要）
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    # API 代理到后台服务
    location /api/ {
        proxy_pass http://127.0.0.1:9998;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
    
    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|avif|webp)$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
}
```

#### 步骤 1：创建 PM2 配置文件

```bash
cd /www/firefly/admin
touch ecosystem.config.js
```

#### 步骤 2：写入配置

```javascript
module.exports = {
  apps: [
    {
      name: 'firefly-admin',
      script: './dist/app.js',
      cwd: '/www/firefly/admin',
      env: {
        NODE_ENV: 'production',
        ADMIN_PORT: '9998',
        BLOG_PATH: '/www/firefly',
      },
      instances: 1,
      autorestart: true,
      watch: false,
      max_memory_restart: '1G',
      error_file: '/www/firefly/admin/logs/error.log',
      out_file: '/www/firefly/admin/logs/output.log',
      log_date_format: 'YYYY-MM-DD HH:mm:ss',
    },
  ],

```

#### 步骤 3：启动后台服务

```bash

npm run build

# 启动 PM2
pm2 start ecosystem.config.js

# 设置开机自启

```

## 2. 更新机制

### 2.1 手动更新（大版本更新）

适用于主题框架大更新、依赖变更等情况。

```bash
# 1. 进入源码目录
cd /www/firefly

# 2. 拉取最新代码
git pull origin main

# 3. 更新依赖（如有变更）
pnpm install

# 4. 更新后台依赖
cd admin
npm install
npm run build
cd ..

# 5. 重新构建前端
pnpm build

# 6. 复制构建产物
cp -r dist/* /www/wwwroot/firefly/

# 7. 重启后台服务
pm2 restart firefly-admin
```

### 2.2 一键更新（后台点击）

在后台管理面板添加"一键更新"按钮，实现无感更新。

#### 步骤 1：创建部署脚本

创建 `/www/firefly/deploy.sh`：

```bash
#!/bin/bash

# Firefly 一键部署脚本
# 使用方法：bash deploy.sh

set -e

echo "=================================="
echo "  Firefly 一键部署脚本"
echo "=================================="

# 1. 进入项目目录
cd /www/firefly
echo "[1/6] 进入项目目录..."

# 2. 拉取最新代码
echo "[2/6] 拉取最新代码..."
git pull origin main

# 3. 更新依赖
echo "[3/6] 更新前端依赖..."
pnpm install

echo "[3/6] 更新后台依赖..."
cd admin
npm install
npm run build
cd ..

# 4. 构建前端
echo "[4/6] 构建前端..."
pnpm build

# 5. 复制构建产物到 Nginx 目录
echo "[5/6] 部署前端..."
cp -r dist/* /www/wwwroot/firefly/

# 6. 重启后台服务
echo "[6/6] 重启后台服务..."
pm2 restart firefly-admin

echo "=================================="
echo "  部署完成！"
echo "=================================="
```

#### 步骤 2：添加执行权限

```bash
chmod +x /www/firefly/deploy.sh
```

#### 步骤 3：创建后台 API 接口

在 `admin/controllers/systemController.ts` 中添加：

```typescript
import { exec } from 'child_process';
import { promisify } from 'util';

const execAsync = promisify(exec);

const updateBlog = async (req: Request, res: Response) => {
  try {
    const result = await execAsync('/www/firefly/deploy.sh', {
      timeout: 300000, // 5分钟超时
      maxBuffer: 1024 * 1024 * 10, // 10MB 缓冲区
    });
    
    return res.json({
      code: 200,
      message: '更新成功',
      data: {
        stdout: result.stdout,
        stderr: result.stderr,
      },
    });
  } catch (error) {
    const err = error as { stdout?: string; stderr?: string; code?: number };
    return res.json({
      code: 500,
      message: `更新失败: ${err.stderr || err.message}`,
      data: {
        stdout: err.stdout,
        stderr: err.stderr,
      },
    });
  }
};
```

#### 步骤 4：添加路由

在 `admin/routes/system.ts` 中添加：

```typescript
router.post('/update', authMiddleware, updateBlog);
```

#### 步骤 5：添加前端按钮

在后台管理面板添加更新按钮（在系统设置页面）。

### 2.3 自动更新（GitHub Webhook）

当 GitHub 仓库有新提交时，自动触发服务器更新。

#### 步骤 1：在 GitHub 配置 Webhook

1. 进入 GitHub 仓库 → Settings → Webhooks
2. 添加 Webhook：
   - Payload URL: `https://blog.example.com/api/system/webhook`
   - Content type: `application/json`
   - Secret: 设置一个密钥（如 `firefly-webhook-secret`）
   - Events: Just the `push` event

#### 步骤 2：创建 Webhook 接口

在 `admin/controllers/systemController.ts` 中添加：

```typescript
import crypto from 'crypto';

const WEBHOOK_SECRET = process.env.WEBHOOK_SECRET || 'firefly-webhook-secret';

const handleWebhook = async (req: Request, res: Response) => {
  try {
    // 验证签名
    const signature = req.headers['x-hub-signature-256'];
    if (!signature) {
      return res.status(401).json({ message: '未授权' });
    }
    
    const payload = JSON.stringify(req.body);
    const hmac = crypto.createHmac('sha256', WEBHOOK_SECRET);
    const digest = `sha256=${hmac.update(payload).digest('hex')}`;
    
    if (signature !== digest) {
      return res.status(401).json({ message: '签名验证失败' });
    }
    
    // 验证分支
    const ref = req.body.ref;
    if (ref !== 'refs/heads/main') {
      return res.status(200).json({ message: '非主分支，忽略' });
    }
    
    // 异步执行部署脚本
    exec('/www/firefly/deploy.sh', (error, stdout, stderr) => {
      if (error) {
        console.error('Webhook 部署失败:', stderr);
      } else {
        console.log('Webhook 部署成功:', stdout);
      }
    });
    
    return res.status(200).json({ message: '收到更新请求，正在处理' });
  } catch (error) {
    return res.status(500).json({ message: '处理失败' });
  }
};
```

#### 步骤 3：添加路由

在 `admin/routes/system.ts` 中添加：

```typescript
router.post('/webhook', handleWebhook);
```

## 3. 写作工作流

### 3.1 Obsidian 本地写作

#### 步骤 1：安装 Obsidian Git 插件

1. 打开 Obsidian → 设置 → 第三方插件
2. 搜索并安装 "Obsidian Git"
3. 配置插件：
   - 设置仓库路径为博客项目目录
   - 开启自动推送（可选）
   - 设置提交间隔

#### 步骤 2：配置写作环境

在 Obsidian 中打开博客项目目录 `/path/to/Firefly/src/content/posts`：

```
Firefly/
├── src/
│   └── content/
│       ├── posts/          # 博客文章
│       │   ├── guide/      # 指南文章
│       │   └── images/     # 文章图片
│       ├── books/          # 书籍数据
│       ├── movies/         # 影视数据
│       └── ...
```

#### 步骤 3：新建文章

使用命令面板（Ctrl+P）运行 `New Post` 命令，或直接创建 `.md` 文件：

```markdown
---
title: 文章标题
published: 2024-01-01
category: 技术
tags: [Astro, Svelte]
image: /images/cover.jpg
description: 文章描述
draft: false
pinned: false
---

文章内容...
```

### 3.2 GitHub 同步

#### 步骤 1：配置 Git

```bash
# 初始化 Git（如果尚未初始化）
cd /path/to/Firefly
git init

# 添加远程仓库
git remote add origin https://github.com/CuteLeaf/Firefly.git

# 设置用户名和邮箱
git config user.name "Your Name"
git config user.email "your.email@example.com"
```

#### 步骤 2：推送代码

```bash
# 查看变更
git status

# 暂存所有变更
git add .

# 提交变更
git commit -m "feat: 新增文章《xxx》"

# 推送到 GitHub
git push origin main
```

#### 步骤 3：Obsidian Git 自动推送

配置 Obsidian Git 插件实现自动推送：

1. 打开插件设置
2. 设置 "Commit message" 模板：`feat: {{file_name}}`
3. 开启 "Auto commit" 和 "Auto push"
4. 设置 "Auto commit interval" 为 5 分钟

### 3.3 Gitee 镜像

#### 步骤 1：添加 Gitee 远程仓库

```bash
git remote add gitee https://gitee.com/CuteLeaf/Firefly.git
```

#### 步骤 2：推送到 Gitee

```bash
git push gitee main
```

#### 步骤 3：设置自动同步

在 GitHub 中创建 Action 实现自动同步到 Gitee：

创建 `.github/workflows/sync-to-gitee.yml`：

```yaml
name: Sync to Gitee

on:
  push:
    branches: [main]

jobs:
  sync:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Sync to Gitee
        uses: wearerequired/git-mirror-action@master
        env:
          SSH_PRIVATE_KEY: ${{ secrets.GITEE_SSH_KEY }}
        with:
          source-repo: "git@github.com:CuteLeaf/Firefly.git"
          destination-repo: "git@gitee.com:CuteLeaf/Firefly.git"
```

## 4. 更新场景汇总

### 4.1 新增/修改文章

| 场景 | 推荐方式 | 步骤 |
|------|----------|------|
| 本地写作后同步 | **Obsidian Git + Webhook** | 写作 → 自动提交 → 自动推送 → 服务器自动构建 |
| 后台直接编辑 | **后台管理面板** | 登录后台 → 编辑文章 → 保存 → 手动点击更新 |
| 快速发布 | **后台上传 Markdown** | 登录后台 → 上传 .md 文件 → 保存 → 手动点击更新 |

### 4.2 修改配置

| 场景 | 推荐方式 | 步骤 |
|------|----------|------|
| 修改站点配置 | **后台配置管理** | 登录后台 → 修改配置 → 保存 → 手动点击更新 |
| 修改主题配置 | **源码修改 + git push** | 修改配置文件 → git push → 服务器自动构建 |

### 4.3 更新主题代码

| 场景 | 推荐方式 | 步骤 |
|------|----------|------|
| 小更新（修复 Bug） | **git push + Webhook** | 修复代码 → git push → 服务器自动构建 |
| 大更新（新功能） | **手动部署** | git pull → 更新依赖 → 构建 → 部署 |

### 4.4 更新后台代码

| 场景 | 推荐方式 | 步骤 |
|------|----------|------|
| 后台功能更新 | **git push + Webhook** | 更新代码 → git push → 服务器自动构建 → PM2 重启 |
| 后台配置变更 | **手动部署** | git pull → 更新依赖 → 构建 → PM2 重启 |

## 5. 故障排查

### 5.1 构建失败

**问题**：`pnpm build` 失败

**排查步骤**：
```bash
# 查看详细错误
pnpm build 2>&1

# 检查 Node.js 版本
node --version

# 检查依赖是否完整
pnpm install

# 清理缓存重新构建
pnpm store prune
pnpm install
pnpm build
```

### 5.2 后台服务无法启动

**问题**：PM2 显示服务异常

**排查步骤**：
```bash
# 查看日志
pm2 logs firefly-admin

# 检查端口是否被占用
netstat -tlnp | grep 9998

# 手动启动测试
cd /www/firefly/admin
node dist/app.js
```

### 5.3 前端页面空白

**问题**：访问网站显示空白页面

**排查步骤**：
```bash
# 检查构建产物是否存在
ls -la /www/wwwroot/firefly/

# 检查 Nginx 配置
nginx -t

# 检查 Nginx 日志
tail -f /www/wwwlogs/firefly_error.log
```

### 5.4 Git 冲突

**问题**：服务器端修改了文件，本地推送时出现冲突

**解决方案**：
```bash
# 在本地拉取服务器代码
git pull origin main

# 解决冲突后重新提交
git add .
git commit -m "fix: 解决冲突"
git push origin main
```

**预防措施**：
- 建议只在一个地方编辑内容（本地或后台）
- 如果需要在后台编辑，编辑后立即推送回 GitHub

## 📝 总结

### 完整工作流

```
本地写作（Obsidian）
    ↓
git commit + push（自动）
    ↓
GitHub 接收
    ↓
Webhook 触发
    ↓
服务器执行 deploy.sh
    ↓
git pull → 安装依赖 → 构建 → 部署 → 重启服务
    ↓
网站自动更新
```

### 关键文件

| 文件 | 作用 |
|------|------|
| `/www/firefly/deploy.sh` | 一键部署脚本 |
| `/www/firefly/admin/ecosystem.config.js` | PM2 配置 |
| `/www/wwwroot/firefly/` | Nginx 托管的静态文件 |
| `.github/workflows/sync-to-gitee.yml` | GitHub → Gitee 同步 |

### 推荐实践

1. **本地写作**：使用 Obsidian，配合 Obsidian Git 插件自动同步
2. **代码托管**：GitHub 作为主仓库，Gitee 作为镜像
